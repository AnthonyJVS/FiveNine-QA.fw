Get Started

Quick Start

From clone to first test run in under two minutes. 

# Clone the repository
git clone https://github.com/AnthonyJVS/FiveNine-QA.fw
cd FiveNine-QA.fw

# Create virtual environment
python -m venv venv
source venv/bin/activate   # or venv\Scripts\activate on Windows

# Install dependencies
pip install -r requirements.txt
playwright install --with-deps chromium

# Configure environment (copy template, edit as needed)
cp config/environments/.env.example config/environments/.env.dev
# Run all tests
pytest

# Smoke tests only (fast CI check)
pytest -m smoke

# UI tests with verbose output
pytest tests/ui/ -v

# API tests
pytest tests/api/ -v

# Debug mode — headed browser + slow motion
HEADLESS=false pytest tests/ui/test_login.py -v --slow-mo=500

# By feature
pytest -m login
pytest -m cart
pytest -m negative
# Full test suite in container
docker compose up tests

# Smoke tests only
docker compose up smoke-tests

# API tests only
docker compose up api-tests

# Rebuild after code changes
docker compose build --no-cache && docker compose up tests
# Install dashboard dependencies
pip install -r requirements-dashboard.txt

# Launch Streamlit dashboard
streamlit run dashboard/app.py

# From the dashboard you can:
#   • Select test suite (smoke / api / ui / all)
#   • Choose browser and headed/headless mode
#   • Run tests and see real-time output
#   • Browse HTML reports and failure screenshots
Capabilities

Key Features

Everything a production-grade QA framework needs, built in from day one.

📄
Page Object Model
Clean separation between test logic and page interactions. Selector changes stay isolated in one file.
🧪
Fixture Injection
Browser, context, page objects, and API client managed by pytest fixtures. Zero manual setup or teardown.
🌐
Multi-Environment
Switch between dev, staging, and production with a single environment variable. No hardcoded URLs.
🎲
Dynamic Test Data
Faker generates unique names, emails, and passwords per run — tests never collide or depend on stale data.
📸
Auto Screenshots
Failure screenshots captured automatically and attached to reports. No manual debugging needed.
🔄
CI/CD Pipeline
GitHub Actions pipeline with staged execution: smoke → API → UI. Auto-cancel on concurrent pushes.
🐳
Docker Support
One-command containerized test execution. No local browser installation required.
📊
Dual Reporting
Self-contained HTML reports and interactive Allure dashboards. Both generated automatically.
📈
Streamlit Dashboard
Web UI for running tests and browsing results — pick suites, browsers, and view output in real time.
